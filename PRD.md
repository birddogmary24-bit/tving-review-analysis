# TVING 리뷰 분석 시스템 - PRD (Product Requirements Document)

## 1. 프로젝트 개요
TVING 앱의 사용자 경험(UX)을 개선하기 위해 구글 플레이 스토어 및 애플 앱스토어의 리뷰를 자동으로 수집하고, AI(Gemini)를 활용하여 정교하게 분석하는 대시보드 시스템입니다.

## 2. 주요 대상 (Target Audience)
- 티빙 서비스 기획자 및 UX 리서처
- CS 운영팀 및 기술 지원팀
- 서비스 품질 관리(QA) 조직

## 3. 핵심 요구사항 (Key Requirements)
### 3.1. 데이터 수집 및 관리
- **다중 플랫폼 지원**: Google Play Store 및 Apple App Store 리뷰 동시 수집.
- **대량 데이터 처리**: 3,000~4,000개 이상의 대규모 리뷰 데이터를 안정적으로 로드.
- **영구 저장소**: Google Cloud Storage(GCS)를 연동하여 서버 재시작 시에도 데이터 보존.
- **중복 방지**: 유니크 리뷰 ID를 기반으로 중복 수집 차단.

### 3.2. AI 분석 (Gemini 1.5 Flash)
- **감성 분류**: 리뷰를 '칭찬', '불만', '기타'로 자동 분류.
- **세부 사유(Sub-category) 체계화**:
    - **불만**: 플레이어 오류, 광고 관련 불만, 요금 및 결제, 콘텐츠 부족, UI/UX 불편, 앱 안정성 등.
    - **칭찬**: 오리지널 콘텐츠, 화질 및 재생 품질, UI 편리성, 콘텐츠 다양성 등.
- **비용 최적화**: 하루 1회 업데이트 제한 로직 적용 (서버 및 API 비용 절감).

### 3.3. 대시보드 및 UI/UX
- **TVING 스타일 테마**: 다크 모드 기반의 프리미엄 UI 적용.
- **시각화 지표**: 연간 누적 리뷰, 긍정 반응 비율, 개선 필요 항목, 월간 성장세 제공.
- **상세 필터링**: 세부 사유별, 플랫폼별, 검색어 기반 리뷰 탐색.
- **엑셀 내보내기**: 분석 결과를 실무에 즉시 활용할 수 있도록 엑셀 다운로드 기능 제공.

## 4. 기술 스택
- **Frontend/Backend**: Next.js 15 (App Router)
- **Deployment**: Google Cloud Run (Memory: 1Gi 권장)
- **Storage**: Google Cloud Storage (JSON 방식)
- **AI**: Google Generative AI (Gemini API)
- **CI/CD**: GitHub 연동 자동 배포

## 5. 업데이트 이력
- **v1.0**: 기본 수집 및 대시보드 구축.
- **v2.0**: GCS 연동 및 클라우드 배포 최적화.
- **v2.5**: 세부 사유 체계화 (플레이어 오류, 광고 불만 집중 관리).
- **v3.0**: 3,500개 이상 대량 데이터 처리 최적화 및 메모리 증설(1Gi).
